Формы.

Доступные формы

Людям с ограниченными возможностями довольно сложно работать с формами, ведь навигация по страницам это одно, а навигация между полями формы это другое. Поэтому добавление номера для каждого элемента форм является не самой плохой идеей.

Навигация по полям

Для полей нужен механизм для навигации без использования таких устройств как мышь. Для облегчения задачи перехода между элементами используются порядковые номера полей и кнопки доступа к полю, это следующие атрибуты которые могут добавляться к отдельным тегам формы а также к тегу legend:

  • tabindex — атрибут для указания порядкового номера поля.
  • accesskey — атрибут для указания горячей клавиши, при нажатии на которую вместе с Alt Shift (для Windows в Chrome, Firefox) фокус устанавливается на поле с этим атрибутом.

See the Pen Forms acessibility by Aleksey Korovin (@alekskorovin) on CodePen.

Подпись для поля ввода

Каждое поле формы как правло содержит индивидуальную подпись. Эта подпись задаётся тего label с атрибутом for в качестве значения которого указывается значение id поля. Таким образом подпись ассоциируется с соответствующим полем. При клике на подпись с заданным корректным значением атрибута for фокус устанавливается на соответствующее поле ввода.

See the Pen Forms labels by Aleksey Korovin (@alekskorovin) on CodePen.

Если тег label включает в себя поле, то можно не указывать атрибут for и при клике на текст, поле так же будет автоматически в фокусе. При этом так же не нужно указывать атрибут id для полей форм.

See the Pen Forms labels without for by Aleksey Korovin (@alekskorovin) on CodePen.

Наборы полей и легенды

Относящиеся к одной по смысле теме поля можно объединять с помощью тега fieldset.

С помощью вложенного тега legend можно задавать подпись обозначающую к группе полей.

При задании fieldset и legend вокруг полей появляются дополнительные элементы, вид которых можно изменить с помощью CSS.

See the Pen Forms fieldsets by Aleksey Korovin (@alekskorovin) on CodePen.

Элемент optgroup позволяет сгруппировать опции selectbox-а. Обязательным для этого элемента является атрибут label который отображается как заголовок над группой элементов, при этом этот заголовок не может быть выбран.

HTML5 формы

HTML5 существенно расширился дополнительными типами текстовых полей input, несколькими новыми атрибутами и удобным дополнительными элементами.

Поддержка новых типов полей и дополнительных улучшений HTML5 форм начинается с Internet Explorer 10, для 9-й версии IE необходимо использовать полифилы (для атрибута placeholder например) либо понимать что новые типы полей будут вести себя как обычные текстовые поля.

Числа и интервалы
Числа

Простое текстовое поле, которое так же позволяет непосредственно редактировать число или увеличивать/уменьшать значение с заданным шагом (обычно используя стрелки вверх/вниз справа внутри поля если они есть, либо при нажатии вверх/вниз на клавиатуре), можно создать типом number.

Атрибут step может задать с каким шагом будет меняться число при его увеличении/уменьшении — число шага будет прибавляться к числу в поле, либо будет вычитаться из числа в поле.

Так же можно задать минимальное и максимальное значение числа с помощью атрибутов min и max соответственно.

Интервалы

Альтернативой полю для ввода числа как текста, служит элемент
<input type="range">.

По умолчанию он отображается как горизонтальная линия со слайдером в середине. Пользователь может отрегулировать положение ползунка смещая его влево либо вправо, при этом минимальное значение находится слева, максимальное справа. Минимальные и максимальные значения указываются в атрибутах min и max.

See the Pen input type range by Aleksey Korovin (@alekskorovin) on CodePen.

При фокусе на элементе можно менять положение ползунка используя клавиатурные кнопки вверх/вниз либо влево/вправо.

В связи с тем что этот элемент отображается в разных браузерах по разному, рекомендуется использовать JavaScript плагины для создания подобных компонентов в веб-странице, например rangeslider.js либо noUISlider.

Даты и время

Для ввода даты и времени существует несколько типов полей. Если поле поддерживается в браузере (а они поддерживаются далеко не везде http://caniuse.com/#search=date), то пользователь может ввести дату или время в определённом формате, просто вбив значения, либо увеличивая/уменьшая значения недели/дня/часа/минуты или же сделав выбор в выпадающем календаре.

  • datetime — для ввода и даты и времени.
  • date — для ввода даты.
  • month — для ввода месяца.
  • week — для ввода недели.
  • week — для ввода времени.
  • datetime-local — для ввода даты и времени в локальных настройках.

See the Pen input date time by Aleksey Korovin (@alekskorovin) on CodePen.

Эти типы полей стоит использовать в админ-частях с поддержкой только Chrome либо Edge. В Firefox в настоящее время эти поля не поддерживаются и ведут себя как обычные текстовые поля.

Так же имеет смысл использовать эти поля при поддержке мобильных устройств:


Атрибуты step, min, max могут быть использованы для этих типов полей, так же как и CSS псевдоклассы :vaild, :invalid.

В случае кросс-браузерной поддержки для компонента выбора даты рекомендуется использовать JS-плагин, например Pikaday.

Элемент datalist

Этот элемент обладает слабой поддержкой среди браузеров http://caniuse.com/#search=datalist. Значение пунктов списка datalist в элементах option выводятся в виде выпадающей подсказки с предлагаемыми вариантами для заполнения текстового поля.

Для того чтобы это работало для поля input необходимо задать значение атрибута id поля datalist в атрибуте list: 

<input name="country" list="countryName">
<datalist id="countryName">
    ...
</datalist>

При этом, когда пользователь вводит первые буквы пунктов, которые существуют в datalist они выводятся в списке, таким образом происходит фильтрация выводимых пунктов исходя из того что вводит пользователь.

See the Pen Datalist by Aleksey Korovin (@alekskorovin) on CodePen.

Отличие такого поля и datalist от элемета select состоит в том, что несмотря на подсказки, пользователь может вводить в поле что угодно.

Для кроссбраузерной поддержки подобных компонентов рекомендуется использовать специальные плагины, такие например, как Select2.

Дополнительные атрибуты

disabled

Чтобы сделать любой элемент формы выключенным достаточно указать этому элементу атрибут disabled.

При этом пользователь не сможет ввести данные в это поле, выбрать значение и даже отправить форму.

See the Pen disabled elements by Aleksey Korovin (@alekskorovin) on CodePen.

required

Чтобы указать, что поле обязательно к заполнению, необходимо добавить атрибут required. При этом встроенная в браузер валидация будет напоминать посредством сообщений о незаполненных полях. Атрибут поддерживается с IE10 — http://caniuse.com/#search=required.

See the Pen Input required by Aleksey Korovin (@alekskorovin) on CodePen.

pattern

Этот атрибут позволяет указать регулярное выражение на языке JavaScript которым будет проверяться введенное значение. Можно указать определённую валидацию для полей типа text, search, url, tel, email, password.

See the Pen Input pattern by Aleksey Korovin (@alekskorovin) on CodePen.

Атрибуты и элементы HTML5 которые не поддерживаются браузерами соответствуют принципу gracefull degradation — если к примеру атрибут placeholder не поддерживается, то он просто не будет отображаться, так же как и элемент datalist, в то время как в современных браузерах пользователь сможет использовать новые возможности в полной мере.

Таблицы.

Лет десять назад и ранее большая часть сайтов версталась таблицами.

Да и дизайн тех времён отталкивался от возможностей таблиц.

Если посмотреть на код сайта mozilla.org в 1999-м году, то мы увидем что сайт свёрстан на таблицах. Даже в 2006-м году сайт Microsoft был создан с использованием таблиц.

Таблицы просты в понимании, но их огромный недостаток в том, что они не отрисовываются браузером, пока не будут просчитаны все строки и стлобцы таблицы, и лишь затем таблица сразу отображается. В то же время вёрстка на div-ах или блочная вёрстка позволяет браузеру отрисовывать блоки, по мере нахождения их в коде и сразу же применять стили, по мере нахождения соответствующего стиля.

В настоящее время сайты верстают блоками, а таблицы используются для табличных данных и и email-шаблонов.

Таблица и CSS

Для таблицы, элемент table можно задавать рамку (border) и внешний отступ (margin), внутренний отступ задать не получится. Так же можно задавть размер (width, height).

Можно задавать отдельно стили для строк и для ячеек таблицы — фоновый цвет и цвет текста, так же шрифт. Отступ можно задавать только внутренний и только для ячеек таблицы td и th.

Рамку (border) можно задавать на все элементы, в случае с tr этот отступ может исчезнуть если так же задаётся отступ на ячейку, рамка на ячейке перекрывает рамку на tr.

По умолчанию фон таблицы и ячеек прозрачный. Если страница или блок, содержащие таблицу, имеют фон, то он будет просвечиваться сквозь таблицу. Если фон задан и для таблицы и для ячеек, то в местах наложения фона таблицы и ячеек будет виден фон только ячеек.

Для ячеек td и th можно задавать размеры, однако в случае ширины, даже при заданном значении, ширина всегда будет рассчитываться относительно ширины всей таблицы.

See the Pen Table columns width depends on the table by Aleksey Korovin (@alekskorovin) on CodePen.

Если не задавать ширину колонкам и ячейкам, они растянутся по контенту и пропорционально количеству колонок.

See the Pen Table columns width depends on the content by Aleksey Korovin (@alekskorovin) on CodePen.

Если задать для таблицы table-layout: fixed то ширина колонок будет браться из первой строки, при этом если контент не влезет, он будет визуально "выпадать" за границы ячеек.

Для любого тега таблицы можно проставлять CSS-класс и ID.

Обнуление стилей для таблицы

Как правило работу с таблицей начинают сбросом стилей по умолчанию: 

table {
    border-collapse: collapse;
    border-spacing: 0;
}

Этот код сбрасывает расстояние между ячейками в 0, и задаёт "схлопывающуюся" модель рамок в таблице — когда рамка смежных ячеек общая, если этого не делать у каждой ячейки будет своя отдельная рамка и будет отступ между ячейками.

Этот код приведен в reset.css.

Для ячеек можно задавать рамку, и внутренний отступ, а так же фон, кроме того можно задавать горизонтальное и вертикальное выравнивание с помощью vertical-align.

Свойство vertical-align задаётся для элементов с моделью inline-block и table-cell может принимать следующие значения:

  • top — выравнивание по верхнему краю.
  • middle — выравнивание по центру.
  • bottom — выравнивание по нижнему краю.

Кроме этого есть значения используемые вместе с inline-block: baseline, sub, super, text-top, text-bottom.

Превращение блочной вёрстки в табличную

С помощью CSS3 блочную вёрстку можно превратить в табличную переопределив значение свойства display.

  • display: table — <table>
  • display: table-row — <tr>
  • display: table-cell — <td>
  • display: table-row-group — <tbody>
  • display: table-header-group — <thead>
  • display: table-footer-group — <tfoot>
  • display: table-column — <col>
  • display: table-column-group — <colgroup>

See the Pen Table with div and styles by Aleksey Korovin (@alekskorovin) on CodePen.
